Introduction : Étude de cas

Cas d'une école

Une école gère les informations suivantes à l'aide de fichiers séparés :

  • 📁 Fichier des étudiants
  • 📁 Fichier des enseignants
  • 📁 Fichier des notes
  • 📁 Fichier des absences

Chaque service stocke ses données indépendamment.

❓ Problématique : Cette méthode est-elle efficace et fiable ?

Inconvénients du système de fichiers

1️⃣ Redondance

Les mêmes données sont enregistrées plusieurs fois.

2️⃣ Incohérence

Une modification dans un fichier ne se reflète pas ailleurs.

3️⃣ Difficulté d'accès

Recherche lente et gestion complexe.

4️⃣ Sécurité limitée

Risque de perte, suppression ou modification non contrôlée.

👉 Il faut donc une solution centralisée : la Base de Données.

Définition d'une Base de Données

Une Base de Données est un ensemble organisé de données stockées de manière structurée et cohérente, permettant un accès rapide, sécurisé et efficace.

Données structurées

Organisées en tables, lignes et colonnes

Accès rapide

Recherches optimisées avec des index

Elle est gérée par un logiciel appelé Système de Gestion de Base de Données (SGBD).

Pourquoi structurer les données ?

Éviter la redondance
Garantir la cohérence
Améliorer la sécurité
Faciliter la recherche
Optimiser la performance
Permettre le partage multi-utilisateur

📌 En résumé :

Une base de données = une seule source de vérité, accessible et fiable !

Ils répondent à la question : "Que doit faire le système ?"

Ils répondent à la question : "Comment le système doit fonctionner ?"

Qu'est-ce que le MCD ?

📌 Définition

Le Modèle Conceptuel de Données (MCD) représente la structure des données indépendamment de toute technologie.

Il décrit :

  • Les entités (objets du monde réel)
  • Les propriétés (attributs)
  • Les associations (relations)

🎯 Objectifs

  • Comprendre et formaliser les besoins
  • Créer un langage commun entre analystes et utilisateurs
  • Faciliter la maintenance et l'évolution

À retenir : Le MCD répond à la question "QUELLES données stocke-t-on ?"

Les Entités

Une entité représente un objet ou un concept du monde réel que l'on souhaite stocker dans la base de données.

  • Pas d'espaces ni de caractères spéciaux

Les Associations

Une association représente un lien entre deux ou plusieurs entités.

1..1

Un à un

Ex: 1 étudiant a 1 dossier

1..N

Un à plusieurs

Ex: 1 classe a plusieurs étudiants

N..N

Plusieurs à plusieurs

Ex: des étudiants suivent plusieurs cours

📌 Cardinalités :

Les cardinalités indiquent le nombre de fois qu'une entité participe à une association.

Exemple : (1,1) = participe exactement 1 fois | (0,N) = peut participer 0, 1 ou plusieurs fois

Exemple : Association (1,N)

🏫 CLASSE → ÉTUDIANT

Une classe contient plusieurs étudiants

CLASSE

(1,1) (0,N)

ÉTUDIANT

1,N

📊 Traduction :

  • Une classe peut avoir plusieurs étudiants
  • Un étudiant appartient à une seule classe

Exemple : Association (N,N)

📚 ÉTUDIANT ↔ COURS

Un étudiant suit plusieurs cours

Un cours est suivi par plusieurs étudiants

ÉTUDIANT

(0,N) (0,N)

COURS

N,N

📝 Traduction :

  • Un étudiant suit plusieurs cours
  • Un cours est suivi par plusieurs étudiants

Attention : Une association N,N deviendra une table à part dans le MLD.

📌 Entité

  • • Représente un objet du monde réel
  • • Nom au singulier, en majuscules
  • • Contient des attributs
  • • A un identifiant unique (clé)

📌 Attribut

  • • Propriété d'une entité
  • • L'identifiant est souligné
  • • A un type (texte, nombre, date)

Introduction au MLD

Le Modèle Logique de Données (MLD) est la transformation du MCD en un modèle compréhensible par un SGBD (Système de Gestion de Base de Données).

MCD

Concepts

Entités, associations

→ MLD ←

Logique

Tables, clés, relations

MPD

Physique

SQL, types précis

📌 À retenir :

Le MLD est indépendant du SGBD mais dépend du modèle relationnel. C'est une étape de traduction du MCD.

Objectifs du MLD

1 Traduire le MCD

Transformer les entités et associations en tables et relations compréhensibles par un SGBD.

2 Optimiser la structure

Éliminer les redondances et préparer un schéma efficace pour l'implémentation.

3 Définir les clés

Identifier les clés primaires (PK) et les clés étrangères (FK) pour assurer l'intégrité des données.

4 Préparer le SQL

Obtenir un schéma directement utilisable pour écrire les instructions CREATE TABLE.

Qu'est-ce qu'un SGBD ?

📌 Définition

Un Système de Gestion de Base de Données (SGBD) est un logiciel qui permet de :

  • Créer et structurer une base de données
  • Manipuler les données (ajout, modification, suppression)
  • Interroger les données (recherches)
  • Gérer les accès et la sécurité

🎯 Exemples de SGBD

MySQL est un SGBD relationnel open source, très utilisé dans le développement web.

Introduction à SQL

SQL (Structured Query Language) est le langage standard pour communiquer avec les bases de données relationnelles.

LDD

Langage de Définition de Données

CREATE, ALTER, DROP

LMD

Langage de Manipulation de Données

INSERT, UPDATE, DELETE

LID

Langage d'Interrogation de Données

SELECT

💡 À retenir :

SQL permet de créer des tables (LDD), d'y insérer/modifier des données (LMD) et de les interroger (LID).

LDD : Création d'une base

📌 CREATE DATABASE

Crée une nouvelle base de données.

CREATE DATABASE nom_base;

-- Exemple
CREATE DATABASE ecole;

📌 USE et DROP

Utiliser et supprimer une base.

-- Utiliser une base
USE ecole;

-- Supprimer une base
DROP DATABASE ecole;

-- Vérifier les bases
SHOW DATABASES;

⚠️ Attention :

DROP DATABASE supprime définitivement toutes les données. À utiliser avec précaution !

LDD : Création de table

Syntaxe CREATE TABLE

CREATE TABLE nom_table (
    nom_colonne1 type [contraintes],
    nom_colonne2 type [contraintes],
    ...
);

Exemple : Création de tables

Table ÉTUDIANT

CREATE TABLE etudiant (
    num_etudiant INT PRIMARY KEY AUTO_INCREMENT,
    nom VARCHAR(50) NOT NULL,
    prenom VARCHAR(50) NOT NULL,
    date_naissance DATE,
    email VARCHAR(100) UNIQUE,
    num_classe INT,
    date_inscription DATETIME DEFAULT CURRENT_TIMESTAMP
);

Table CLASSE

CREATE TABLE classe (
    num_classe INT PRIMARY KEY AUTO_INCREMENT,
    nom_classe VARCHAR(50) NOT NULL UNIQUE,
    niveau VARCHAR(20),
    effectif INT DEFAULT 0,
    CHECK (effectif >= 0)
);

🔗 Ajout de la clé étrangère :

ALTER TABLE etudiant 
ADD CONSTRAINT fk_classe 
FOREIGN KEY (num_classe) 
REFERENCES classe(num_classe);

LDD : Modifier une table (ALTER)

📝 Ajouter une colonne

ALTER TABLE etudiant ADD adresse VARCHAR(255);

✏️ Modifier une colonne

ALTER TABLE etudiant MODIFY nom VARCHAR(100) NOT NULL;

🗑️ Supprimer une colonne

ALTER TABLE etudiant DROP COLUMN adresse;

📌 Renommer une colonne

ALTER TABLE etudiant RENAME COLUMN nom TO nom_famille;

LDD : Supprimer des tables

DROP TABLE

Supprime complètement une table (structure et données).

DROP TABLE etudiant;

-- Éviter erreur si table n'existe pas
DROP TABLE IF EXISTS etudiant;

TRUNCATE TABLE

Vide la table mais garde la structure.

TRUNCATE TABLE etudiant;

⚠️ Différence importante :

  • DROP : supprime la table (elle n'existe plus)
  • TRUNCATE : vide la table (elle existe toujours, mais sans données)

LMD : Insérer des données (INSERT)

📝 Insertion simple

-- Spécifier les colonnes
INSERT INTO classe (nom_classe, niveau) 
VALUES ('BTS SIO 1', 'BTS');

-- Toutes les colonnes
INSERT INTO classe 
VALUES (1, 'BTS SIO 1', 'BTS', 25);

📝 Insertion multiple

INSERT INTO etudiant (nom, prenom, email) 
VALUES 
    ('Dupont', 'Jean', 'jean@email.com'),
    ('Martin', 'Marie', 'marie@email.com'),
    ('Bernard', 'Pierre', 'pierre@email.com');

💡 Astuce :

Avec AUTO_INCREMENT, ne pas spécifier la valeur, elle sera automatique.

LMD : Interroger les données (SELECT)

Syntaxe de base

SELECT colonnes
FROM table
[WHERE condition]
[ORDER BY colonne]
[LIMIT n];

Exemples simples :

-- Toutes les colonnes
SELECT * FROM etudiant;

-- Colonnes spécifiques
SELECT nom, prenom FROM etudiant;

-- Avec alias
SELECT nom AS 'Nom de famille' FROM etudiant;

-- Distinct
SELECT DISTINCT nom FROM etudiant;

Avec conditions :

-- WHERE
SELECT * FROM etudiant 
WHERE nom = 'Dupont';

-- Opérateurs
SELECT * FROM etudiant 
WHERE age >= 18 AND ville = 'Paris';

-- LIKE
SELECT * FROM etudiant 
WHERE email LIKE '%@gmail.com';

LMD : Modifier des données (UPDATE)

Syntaxe

UPDATE table
SET colonne1 = valeur1, colonne2 = valeur2
WHERE condition;

⚠️ IMPORTANT

Toujours mettre un WHERE, sinon TOUTES les lignes seront modifiées !

Exemples

-- Modifier un étudiant
UPDATE etudiant 
SET email = 'nouveau@email.com' 
WHERE num_etudiant = 1;

-- Modifier plusieurs colonnes
UPDATE etudiant 
SET nom = 'DUPONT', prenom = 'JEAN' 
WHERE num_etudiant = 2;

-- Avec calcul
UPDATE produit 
SET prix = prix * 1.10 
WHERE categorie = 'informatique';

LMD : Supprimer des données (DELETE)

DELETE

DELETE FROM table WHERE condition;

-- Supprimer un étudiant
DELETE FROM etudiant 
WHERE num_etudiant = 3;

-- Supprimer plusieurs
DELETE FROM etudiant 
WHERE classe = 'BTS 1';

-- Supprimer tout (mais garde structure)
DELETE FROM etudiant;

⚠️ Attention :

DELETE sans WHERE supprime TOUTES les lignes !

Comparaison

CommandeEffetRapide?
DELETESupprime ligne par ligneLent
TRUNCATEVide la tableRapide
DROPSupprime la tableRapide

Pour vider une table : TRUNCATE est plus rapide que DELETE sans WHERE.

Les jointures (JOIN)

Les jointures permettent d'interroger plusieurs tables à la fois.

INNER JOIN

SELECT e.nom, e.prenom, c.nom_classe
FROM etudiant e
INNER JOIN classe c 
    ON e.num_classe = c.num_classe;

LEFT JOIN

SELECT e.nom, c.nom_classe
FROM etudiant e
LEFT JOIN classe c 
    ON e.num_classe = c.num_classe;

📌 Explication :

  • INNER JOIN : uniquement les correspondances
  • LEFT JOIN : tous les étudiants, même sans classe
  • RIGHT JOIN : toutes les classes, même vides

Fonctions d'agrégation

FonctionDescription
COUNT()Nombre de lignes
SUM()Somme
AVG()Moyenne
MAX()Valeur maximale
MIN()Valeur minimale
-- Nombre d'étudiants
SELECT COUNT(*) FROM etudiant;

-- Moyenne des notes par cours
SELECT code_cours, AVG(note) as moyenne
FROM suivre
GROUP BY code_cours;

-- Max et Min
SELECT MAX(prix), MIN(prix) FROM produit;

-- Avec condition
SELECT classe, COUNT(*) as effectif
FROM etudiant
GROUP BY classe
HAVING COUNT(*) > 10;

💡 GROUP BY : regroupe les lignes | HAVING : filtre les groupes (comme WHERE pour GROUP BY)

Exercice SQL n°2 : Requêtes

📌 Requêtes sur la bibliothèque

Écrivez les requêtes SQL pour :

  1. Afficher tous les livres
  2. Afficher les livres disponibles
  3. Afficher les emprunts en cours (non retournés)
  4. Afficher les emprunts avec les noms des membres et titres des livres
  5. Compter le nombre d'emprunts par membre
  6. Trouver le membre qui a emprunté le plus de livres
  7. Mettre à jour le statut d'un emprunt (rendu = TRUE)
  8. Supprimer un membre (et ses emprunts ?)

💡 Indice :

Pensez aux jointures (JOIN), aux fonctions d'agrégation (COUNT, MAX) et à WHERE.

Correction : Requêtes bibliothèque 

-- 1. Afficher tous les livres
SELECT * FROM livre;

-- 2. Afficher les livres disponibles
SELECT * FROM livre WHERE disponible = TRUE;

-- 3. Afficher les emprunts en cours
SELECT * FROM emprunt WHERE rendu = FALSE;

-- 4. Emprunts avec noms et titres
SELECT e.num_emprunt, m.nom, m.prenom, l.titre, e.date_emprunt
FROM emprunt e
JOIN membre m ON e.num_membre = m.num_membre
JOIN livre l ON e.isbn = l.isbn
WHERE e.rendu = FALSE;

-- 5. Compter les emprunts par membre
SELECT m.nom, m.prenom, COUNT(e.num_emprunt) as nb_emprunts
FROM membre m
LEFT JOIN emprunt e ON m.num_membre = e.num_membre
GROUP BY m.num_membre;

-- 6. Membre qui a emprunté le plus
SELECT m.nom, m.prenom, COUNT(e.num_emprunt) as nb_emprunts
FROM membre m
JOIN emprunt e ON m.num_membre = e.num_membre
GROUP BY m.num_membre
ORDER BY nb_emprunts DESC
LIMIT 1;

-- 7. Mettre à jour un emprunt (retour)
UPDATE emprunt SET rendu = TRUE, date_retour = CURDATE()
WHERE num_emprunt = 1;

-- 8. Supprimer un membre (d'abord ses emprunts)
DELETE FROM emprunt WHERE num_membre = 1;
DELETE FROM membre WHERE num_membre = 1;

Exercice SQL n°3 : Gestion de stock

📦 Gestion de stock

Soit les tables suivantes :

produit (code_produit, nom, prix, stock, seuil_alerte, code_categ)
categorie (code_categ, nom_categ)
fournisseur (num_fournisseur, nom, telephone)
commande (num_commande, date_commande, num_fournisseur)
ligne_commande (num_commande, code_produit, quantite, prix_achat)

Écrivez les requêtes pour :

  1. Créer les tables avec les contraintes
  2. Afficher les produits en stock faible (stock < seuil_alerte)
  3. Afficher le nombre de produits par catégorie
  4. Afficher les commandes d'un fournisseur
  5. Augmenter tous les prix de 5%
  6. Supprimer un produit (vérifier les dépendances)

Correction : Gestion de stock 

-- 1. Création des tables
CREATE TABLE categorie (
    code_categ INT PRIMARY KEY AUTO_INCREMENT,
    nom_categ VARCHAR(50) NOT NULL UNIQUE
);

CREATE TABLE produit (
    code_produit INT PRIMARY KEY AUTO_INCREMENT,
    nom VARCHAR(100) NOT NULL,
    prix DECIMAL(10,2) CHECK (prix >= 0),
    stock INT DEFAULT 0,
    seuil_alerte INT DEFAULT 5,
    code_categ INT,
    FOREIGN KEY (code_categ) REFERENCES categorie(code_categ)
);

CREATE TABLE fournisseur (
    num_fournisseur INT PRIMARY KEY AUTO_INCREMENT,
    nom VARCHAR(100) NOT NULL,
    telephone VARCHAR(20)
);

CREATE TABLE commande (
    num_commande INT PRIMARY KEY AUTO_INCREMENT,
    date_commande DATETIME DEFAULT CURRENT_TIMESTAMP,
    num_fournisseur INT,
    FOREIGN KEY (num_fournisseur) REFERENCES fournisseur(num_fournisseur)
);

CREATE TABLE ligne_commande (
    num_commande INT,
    code_produit INT,
    quantite INT CHECK (quantite > 0),
    prix_achat DECIMAL(10,2),
    PRIMARY KEY (num_commande, code_produit),
    FOREIGN KEY (num_commande) REFERENCES commande(num_commande),
    FOREIGN KEY (code_produit) REFERENCES produit(code_produit)
);

-- 2. Produits en stock faible
SELECT * FROM produit WHERE stock < seuil_alerte;

-- 3. Nombre de produits par catégorie
SELECT c.nom_categ, COUNT(p.code_produit) as nb_produits
FROM categorie c
LEFT JOIN produit p ON c.code_categ = p.code_categ
GROUP BY c.code_categ;

-- 4. Commandes d'un fournisseur
SELECT c.num_commande, c.date_commande, p.nom, lc.quantite
FROM commande c
JOIN ligne_commande lc ON c.num_commande = lc.num_commande
JOIN produit p ON lc.code_produit = p.code_produit
WHERE c.num_fournisseur = 1;

-- 5. Augmenter tous les prix de 5%
UPDATE produit SET prix = prix * 1.05;

-- 6. Supprimer un produit (d'abord les lignes de commande)
DELETE FROM ligne_commande WHERE code_produit = 1;
DELETE FROM produit WHERE code_produit = 1;
